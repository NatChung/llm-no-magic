Set mode to "raw prompt", leave System empty. User:

一年有幾個月?

Step 2 · Switch to "product processing"

Keep the same User text, switch to "product processing" and send. Expand the preview — the final prompt now becomes:

<|im_start|>user
一年有幾個月?<|im_end|>
<|im_start|>assistant
<think>
</think>

爸爸有3顆蘋果,兒子多他2顆。請問兒子幾顆?

Step 1 · Direct answer (skip thinking)

Set mode to "direct answer" and send. Look at the preview — the final prompt ends with <think></think>. I'm stuffing that in to trick the model into thinking it's already done thinking.

Step 2 · Use thinking

Switch to "use thinking" and send. In the preview — the tail no longer has a stuffed think block; it's left empty so the model can think for itself.

[Agent class] Phase 1 — Agent loop warm-up

5 min

Run in Terminal (explicitly cd back to the project root, because Phase 0 left the shell sitting in /tmp/wow):

cd ~/projects/llm-no-magic && python3 -m agent.agent

At the prompt, type:

現在幾點?

"On Wednesday we used the 0.6B model in Tabs ①②③. Today Tab ④ is 4B. It's bigger, but the mechanism is exactly the same — it still spits out tokens, still has a probability distribution, still takes context in and we watch how the distribution shifts.

The only difference: among the tokens it emits, there are <tool_call>, "name", "arguments" — trigger patterns the client recognizes. The client sees them → actually runs the tool → puts the result back into the prompt → the model spits more tokens → repeat, until the model stops triggering and gives a final answer.

That's it. That's an agent. No magic, just tokens plus a convention."

[Agent class] Phase 2 — Read + write Core

12–15 min

Step 1 · Open Tab ④ Agent

In the browser, open the LLM workshop site → click Tab ④ Agent:

http://localhost:9000/./frontend/

Step 2 · Project the System prompt for the audience

The System prompt textarea is pre-filled. Project it on screen and read it through once:

You are a helpful assistant with access to tools…

"This is the client telling the model what tools it has available."

Step 3 · Pick preset 2 "Read + write summary"

Click the preset menu → pick preset 2 "Read + write summary" → the textarea auto-fills with:

讀 prompts.md,把它總結成 3 點,寫到 ~/Desktop/llm-summary.md

Press ▶ Run agent.

Step 4 · Narration — Turn 1

  • "After you press the button, you'll see the model think for a second, then the Turn 1 block shows up."
  • "That string at the top of Turn 1 is the token stream the model emitted. Look at the very first token — it's <tool_call>."
  • "Click it once and look at the bottom right: 99.9%. Every other option is below 0.1%."
  • "Which means: after reading the system prompt plus the question, the model is essentially 100% sure it wants to take the tool path."
  • "Click name, ":", "read_file" — every token shows its top-10 at that moment."
  • "llama parses read_file({\"path\":\"prompts.md\"}) out of that raw token stream, our backend actually reads that file, and the result lands in the green box."

Step 5 · Narration — Turn 2 → Turn 3

  • "Then into Turn 2 — the model sees the file content that came back, decides the next move is write_file, same flow."
  • "In Turn 3 the model finally stops calling tools and gives a final answer — and look, llm-summary.md is actually sitting on the Desktop now."

Step 6 · Confirm the file is real

Open Finder, or check in the terminal:

cat ~/Desktop/llm-summary.md && open ~/Desktop

[Agent class] Phase 3 — Exec runs a script

8–10 min

Step 1 · Pick preset 3 "Count the .md files"

In Tab ④ Agent, pick preset 3. The auto-filled prompt:

數一下這個 repo 底下有幾個 .md 檔

Press ▶ Run agent.

Step 2 · Narration

  • "This time the model doesn't read a file, doesn't write a file — it picks exec_bash."
  • "Click the <tool_call> token, check the probability — same story, 99%+."
  • "The parsed tool_call shows exec_bash({\"command\":\"find . -name '*.md' | wc -l\"}) (or some similar find / ls command) — the model wrote that shell command itself."
  • "In the green box you see a real exit=0, and stdout is a number."
  • "The final answer is just that number."

[Agent class] Phase 4 — Skill / MCP / API hint

5 min

Talk through the concepts (no demo — that goes in V3 advanced):

  • Skill = a package of tools (prompt + tools bundled together)
  • MCP = a protocol for tools (run tools on an external server)
  • API = the reach of tools (connect to the internet)
