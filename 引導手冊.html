[Agent 課] Phase 1 — Agent loop 暖身

5 min

Terminal 跑(明確 cd 回 project root,因 Phase 0 把 shell 帶到 /tmp/wow):

cd ~/projects/llm-no-magic && python3 -m agent.agent

Prompt 打:

現在幾點?

「Wed 課我們在 Tab ①②③ 看的是 0.6B,今天 Tab ④ 是 4B。比較大,但機制完全一樣 — 還是吐 token、還是有機率分佈、還是 context 餵進去看分佈怎麼變。

差別只在:吐出來的 token 裡有 <tool_call>、"name"、"arguments" 這些 client 看得懂的 trigger pattern,client 認得 → 真的去執行 → 結果回 prompt 再吐 → 直到 model 不再 trigger,給最終答案。

這就是 Agent。沒有魔法,只有 token + 約定。」

[Agent 課] Phase 2 — 讀+寫 核心

12–15 min

Step 1 · 打開 Tab ④ Agent

瀏覽器開 LLM 課 site → 點 Tab ④ Agent:

http://localhost:9000/./frontend/

Step 2 · System prompt 投影給觀眾

System prompt textarea 已預填,把它投影出來念一次:

You are a helpful assistant with access to tools…

「這是 client 教 model 它有什麼能用。」

Step 3 · 選 preset 2「讀+寫 摘要」

點 preset 選單 → 選 preset 2「讀+寫 摘要」 → textarea 自動填入:

讀 prompts.md,把它總結成 3 點,寫到 ~/Desktop/llm-summary.md

按 ▶ Run agent。

Step 4 · Narration — Turn 1

  • 「按下去之後,你會看到 model 思考一秒,然後 Turn 1 block 出現」
  • 「Turn 1 上面這串就是 model 吐的 token,注意第一個 token — 是 <tool_call>」
  • 「點它一下,看右下角:99.9%。剩下的選項機率小於千分之一」
  • 「意思是 model 看完 system prompt + 問題之後,幾乎 100% 確定要走 tool 那條路」
  • 「再點 name、":"、"read_file" 這些 token — 都看得到當下 top-10」
  • 「llama 從這串 raw token 解析出 read_file({"path":"prompts.md"}),我們 backend 真的去讀那個檔案,結果在綠色框」

Step 5 · Narration — Turn 2 → Turn 3

  • 「然後進 Turn 2 — model 看到讀回來的內容,決定下一步要 write_file,同樣的流程」
  • 「Turn 3 最終 model 不再 tool_call,給最終答案 — 你看 Desktop 上 llm-summary.md 真的被寫出來」

Step 6 · 確認真檔案

打開 Finder / 在 terminal 確認:

cat ~/Desktop/llm-summary.md && open ~/Desktop

[Agent 課] Phase 3 — Exec 跑腳本

8–10 min

Step 1 · 選 preset 3「數 .md 檔」

Tab ④ Agent 裡選 preset 3。自動填入的 prompt:

數一下這個 repo 底下有幾個 .md 檔

按 ▶ Run agent。

Step 2 · Narration

  • 「這次 model 不讀檔、不寫檔 — 它選 exec_bash」
  • 「點 <tool_call> token 看機率 — 一樣 99%+」
  • 「parsed tool_call 顯示 exec_bash({"command":"find . -name '*.md' | wc -l"})(或類似的 find / ls cmd)— model 自己生 shell 指令」
  • 「綠色框看到真的 exit=0、stdout 是數字」
  • 「Final answer 就是那個數字」

[Agent 課] Phase 4 — Skill / MCP / API hint

5 min

  • Skill = 工具的 package(prompt + tools 綁一起)
  • MCP = 工具的協議(去外部 server 跑 tool)
  • API = 工具的 reach(連 internet)
